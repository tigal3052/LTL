# Reference: 슬롯머신식 보상과 변동 강화 분석

## 확인한 웹 근거

- 강화 스케줄 이론은 보상이 언제 주어지는지를 정하는 규칙이며, 부분 강화는 고정/변동, 간격/비율 조합으로 나뉜다.  
  출처: https://www.simplypsychology.org/schedules-of-reinforcement.html
- 변동 비율 강화는 예측할 수 없는 반응 횟수 뒤에 보상이 주어지는 구조로, 높은 지속 반응을 유도하는 것으로 설명된다.  
  참고 출처: https://www.verywellmind.com/what-is-a-variable-ratio-schedule-2796012

## 핵심 관찰

슬롯머신식 보상은 단순히 랜덤 보상을 뜻하지 않습니다. 중요한 것은 `곧 나올 것 같은 기대`, `결과 공개 전의 지연`, `작은 손실 뒤에도 다음 시도를 상상하게 하는 구조`입니다. LTL은 이 원리를 과금/착취가 아니라 노드 클리어 보상, 유물 후보, 빌드 실험 동기에 제한적으로 사용해야 합니다.

## 작동 원리

1. **변동 비율 기대**  
   매번 같은 보상이 아니라 1~5개 후보, 희귀도, 현재 빌드 관련성이 흔들린다.

2. **예측 가능한 바닥과 예측 불가능한 천장**  
   플레이어는 최소 보상이 있다는 안정감을 갖고, 동시에 더 좋은 결과를 기대한다.

3. **결과 전 연출 지연**  
   너무 즉시 공개하면 기대감이 사라지고, 너무 길면 피로해진다. 짧은 고조가 적절하다.

4. **근접 실패 효과의 절제**  
   “거의 전설이었다” 같은 과도한 near-miss 연출은 피로와 불신을 만든다. LTL은 실패를 조작처럼 보이게 하면 안 된다.

5. **빌드 관련성**  
   완전 랜덤보다 현재 파동/백팩 상태와 약하게 연결된 랜덤이 더 좋은 선택 고민을 만든다.

## LTL 적용 방침

- 노드 클리어 보상 수량은 1~5개 가중치 테이블을 사용한다.
- 최소 1개는 현재 빌드와 관련된 역할 또는 파동 후보가 되게 한다.
- 희귀도는 stage index, node risk, 기존 유물 태그와 연동한다.
- 보상 공개 전 연출은 짧고 건조하게 반복 가능해야 한다.
- 실패 런에는 작은 확정 보상 또는 지식/로그 피드백을 제공한다.

## 금지선

- 유료 재시도, 유료 reroll, 확률 은폐는 하네스 범위에서 금지한다.
- near-miss를 과도하게 연출하지 않는다.
- 보상 결과를 조작했다는 느낌을 주지 않도록 seed/replay 로그에 후보 생성 결과를 남긴다.
- 보상을 너무 자주 터뜨려 전투의 파동 판독을 흐리지 않는다.

## 구현 방침

```text
RewardResolver
  input: seed, stage_index, node_type, node_weakness_tags, backpack_tags
  output: reward_count, candidate_items[], rarity_rolls[], reason_tags[]
```

필수 로그:

- `reward_roll_started`
- `reward_count_resolved`
- `reward_candidate_generated`
- `reward_revealed`
- `reward_selected`

필수 지표:

- 평균 보상 수량
- 희귀도 분포
- 현재 빌드 관련 후보 비율
- reroll 또는 포기율
- 보상 선택 후 다음 노드 클리어율 변화

## 검증 질문

- 보상 후보가 완전 랜덤처럼 느껴지지 않고, 현재 빌드와 대화하는가?
- 좋은 보상을 얻지 못해도 다음 노드 선택 전략이 남는가?
- 보상 공개 연출이 반복 플레이에서 지루하지 않은가?
