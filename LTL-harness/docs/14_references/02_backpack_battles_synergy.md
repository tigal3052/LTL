# Reference: Backpack Battles의 공간 시너지 분석

## 확인한 웹 근거

- Steam 공식 페이지는 Backpack Battles를 `inventory management auto battler`로 소개하며, 강한 빌드를 위해 아이템 구매와 배치가 중요하다고 설명한다.  
  출처: https://store.steampowered.com/app/2427700/_Backpack_Battles/
- 같은 페이지는 아이템이 서로 다른 모양, 크기, 가격, 희귀도를 가지며, 조합을 통해 더 강한 장비를 만든다고 설명한다.

## 핵심 관찰

Backpack Battles의 재미는 아이템 자체보다 `공간이 곧 규칙`이라는 점에서 나온다. 플레이어는 좋은 아이템을 사는 것만으로 끝나지 않고, 어디에 두어야 효율이 살아나는지를 계속 계산한다.

## 작동 원리

1. **형태가 비용이 된다**  
   큰 아이템은 강하지만 공간을 많이 차지한다. 작은 아이템은 약해도 틈새에 들어가 엔진을 완성한다.

2. **인접이 의미를 만든다**  
   면 접촉, 주변 슬롯, 특정 방향 조건이 아이템 효과를 바꾼다. 배치는 미관이 아니라 전투력 계산이다.

3. **상점과 백팩이 한 루프다**  
   구매 선택은 현재 백팩 공간과 미래 조합 가능성을 동시에 고려하게 만든다.

4. **조합의 예고가 중요하다**  
   플레이어가 “이 아이템을 하나 더 찾으면 완성된다”는 기대를 갖는다. 완성 직전의 미완성 엔진이 동기를 만든다.

5. **상대 빌드/카운터 사고**  
   원작은 PvP/오토배틀이지만 핵심은 `내 공간 엔진이 외부 문제를 어떻게 해결하는가`에 있다.

## LTL에 적용할 것

- 백팩은 전투 전 메뉴가 아니라 전투 리듬 생성기다.
- 아이템은 최소한 `shape`, `tags`, `generated_energy`, `cooldown`, `adjacency_rules`를 가진다.
- 인접 시너지는 쿨타임, 생성 파동, 큐 용량, 수리 안정성, 방해 저항 중 하나를 바꿔야 한다.
- 저주/축복 셀을 통해 고정 정답 배치를 방지한다.
- 희귀 유물은 크거나 까다로운 조건을 가져야 한다. 강한 효과에는 공간적 책임이 필요하다.

## LTL에 그대로 가져오면 안 되는 것

- 순수 오토배틀 결과 관전 구조는 LTL의 수동 채굴 감각과 다르다.
- PvP 카운터 중심 복잡도는 프로토타입 범위에서 제외한다.
- 아이템 수를 초반부터 과도하게 늘리면 파동/큐 검증보다 조합 암기가 앞선다.

## LTL 백팩 구현 규칙

| 요소 | 권장 방침 |
|---|---|
| 셀 크기 | 전투 HUD와 동시에 읽히도록 고정 셀, 최소 44px 이상 기준 |
| 접촉 판정 | 기본은 상하좌우 면 접촉만 인정 |
| 대각선 판정 | 별도 태그가 있는 유물만 허용 |
| 시너지 표시 | 아이템 사이에 얇은 라인/펄스 표시, 텍스트 남발 금지 |
| 미완성 조합 | 조건 부족 상태를 회색 라인 또는 잠긴 아이콘으로 표시 |
| 저주/축복 셀 | 셀 테두리 패턴으로 표시하고 색만으로 구분하지 않음 |

## 프로토타입 최소 아이템 풀

- 채굴기 1종: 기본 발사/내구도 중심
- 빨강/파랑/보라/초록 유물 각 1종
- 동력 장치 1종: 인접 쿨타임 감소
- 수리 도구 1종: 수리 시간 감소 또는 내구도 손실 완화
- 보조 유물 1종: 큐 용량 또는 누수율 개선

## 테스트 기준

- 같은 배치는 같은 seed에서 같은 큐 생성 결과를 만든다.
- 아이템 회전 후 점유 셀과 인접 셀이 정확히 계산된다.
- 인접 시너지 on/off에 따른 지표 차이가 telemetry에 기록된다.
- 시너지 UI가 도메인 계산 결과와 불일치하지 않는다.
