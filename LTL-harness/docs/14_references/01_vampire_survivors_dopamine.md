# Reference: Vampire Survivors의 도파민 루프 분석

## 확인한 웹 근거

- Steam 공식 페이지는 Vampire Survivors를 `time survival`, `minimalistic gameplay`, `roguelite elements`로 설명하며, 선택이 빠른 snowball을 만든다고 소개한다.  
  출처: https://store.steampowered.com/app/1794680/Vampire_Survivors/
- 공식 설명의 시작 팁은 보석과 아이템을 천천히 회수하고, 초반에는 공격 무기 2~3개를 잡되 하나씩 집중 성장시키라고 안내한다. 이는 플레이어가 화면을 이동하며 보상을 회수하고, 성장 선택을 압축적으로 반복하는 구조를 보여준다.

## 핵심 관찰

Vampire Survivors의 강점은 복잡한 입력이 아니라 `자동 전투 + 짧은 선택 + 폭발적 성장 피드백`입니다. 플레이어가 직접 조준하지 않아도 화면 전체가 결과를 계속 보여주고, 경험치 보석/상자/레벨업 선택이 작은 보상과 큰 보상을 교차시킵니다.

## 작동 원리

1. **입력 부담 최소화**  
   플레이어는 주로 이동과 선택에 집중한다. 공격은 자동화되어 있어 결과를 계속 관찰할 수 있다.

2. **작은 보상 밀도**  
   보석, 동전, 적 처치, 숫자 상승이 짧은 간격으로 발생한다. 큰 보상이 없어도 화면은 계속 진행감을 준다.

3. **큰 보상 연출**  
   보물 상자, 레벨업, 진화 무기 같은 순간은 음악/빛/지연 연출로 기대감을 끌어올린다. 핵심은 보상 결과보다 `열리는 과정`이 기억된다는 점이다.

4. **스노우볼 가시성**  
   선택이 누적되면 투사체 수, 범위, 지속시간, 처치 속도가 화면에 바로 드러난다. 플레이어는 빌드가 강해졌다는 것을 설명이 아니라 화면 밀도로 이해한다.

5. **짧은 런 안의 장기 성장**  
   한 판은 실패해도 골드/해금/다음 시도 개선으로 이어진다. 실패가 완전 손실이 아니라 다음 런의 준비가 된다.

## LTL에 적용할 것

- LTL의 전투는 자동 전투가 아니지만, `에너지 큐 생성`은 자동화하고 `발사/조준`만 수동화해 입력 부담을 제한한다.
- 작은 보상은 전투 중 즉시 제공한다: 매칭 피격, 큐 충전, 핀 경고 회피, 비활성 타일 회복, 수리 완료.
- 큰 보상은 노드 클리어에 집중한다: 지형 균열, 황금빛 누출, 사운드 고조, 1~5개 유물 후보 공개.
- 스노우볼은 백팩에서 보여준다: 쿨타임 단축, 큐 생성 색상 변화, 인접 시너지 라인, 유물 활성화 펄스.
- 실패 후에도 telemetry report와 최소 보상을 통해 다음 실험 동기를 만든다.

## LTL에 그대로 가져오면 안 되는 것

- 화면 전체를 적/투사체로 가득 채우는 방식은 LTL의 3x10 파동 판독을 방해한다.
- 자동 공격 중심 구조는 LTL의 수동 조준/발사 핵심을 약화시킨다.
- 보상 빈도를 너무 높이면 4핀 시간 압박과 채굴 집중감이 흐려진다.

## 구현 방침

- 노드 클리어 보상 연출은 3단계로 만든다: `균열 발생 -> 빛/음향 고조 -> 유물 후보 등장`.
- 보상 후보가 나타나기 전 0.6~1.2초의 짧은 기대 지연을 둔다.
- 상자식 연출은 노드 클리어에만 사용하고, 일반 피격에는 짧은 타격/파동 피드백만 사용한다.
- 로그 지표로 `clear_reward_count`, `reward_reveal_duration_sec`, `reward_selected_id`를 기록한다.

## 검증 질문

- 플레이어가 노드 클리어 직전부터 보상 기대감을 느끼는가?
- 보상 후보 공개가 다음 백팩 배치 고민으로 이어지는가?
- 전투 중 이펙트가 파동 색상 판독을 방해하지 않는가?
