# Inventory & Synergy Specs

## 역할

백팩은 전투 전 장비 보관함이 아니라 실시간 채굴 엔진입니다. 배치에 따라 쿨타임, 생성 파동, 큐 충전량, 수리 안정성이 달라져야 합니다.

## 기본 규칙

- 아이템은 폴리오미노 형태를 가진다.
- 아이템은 `artifact`, `drill`, `power`, `repair`, `support` 같은 역할 태그를 가진다.
- 회전 가능 여부는 아이템 데이터에 명시한다.
- 동일 셀 중첩은 금지한다.
- 배치 유효성은 `InventoryModel`의 순수 함수로 판정한다.

## 인접 시너지

| 시너지 | 조건 | 효과 예시 |
|---|---|---|
| 동력 장치 인접 | artifact가 power와 면으로 접함 | 쿨타임 감소 |
| 수리 도구 인접 | repair가 drill/support와 면으로 접함 | 내구도 손실 회복 또는 수리 시간 감소 |
| 같은 파동 공명 | 같은 pulse tag 아이템 2개 이상 인접 | 해당 색 에너지 생성량 증가 |
| 보조 유물 링크 | support가 artifact 2개 이상과 인접 | 큐 용량 또는 생성 안정성 증가 |

대각선 접촉은 기본 시너지로 보지 않습니다. 대각선 시너지는 별도 태그가 있을 때만 허용합니다.

## 욕심의 비용

- 당장 작동하지 않는 강력한 유물을 들고 가면 공간과 시너지 면을 차지한다.
- 저주/축복 룬은 특정 백팩 셀에 영구 버프/너프 슬롯을 부여해 정답 배치 고착화를 방지한다.
- 프로토타입에서는 6~10개 아이템과 2~3종 시너지만 구현해 체감 검증에 집중한다.

## 데이터 필드 초안

```json
{
  "id": "artifact_red_core",
  "display_name": "붉은 핵",
  "rarity": "common",
  "tags": ["artifact", "pulse_red"],
  "shape": [[0, 0], [1, 0]],
  "rotatable": true,
  "base_cooldown_sec": 1.2,
  "generated_energy": "red",
  "synergy_rules": ["adjacent_power_cooldown_down"]
}
```

## 테스트 대상

- 회전/배치 유효성
- 인접 면 계산
- 같은 seed의 시작 아이템/보상 후보 재현성
- 시너지 적용 전후 쿨타임/생성량 변화
- 저주 셀에 배치했을 때 페널티 적용
