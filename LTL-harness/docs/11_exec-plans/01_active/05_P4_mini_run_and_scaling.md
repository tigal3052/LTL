# P4. 미니 런과 스케일링

## 목표

단일 노드 전투를 넘어 3스테이지 미니 런에서 성장, 보상, 노드 선택, 난이도 상승이 함께 유지되는지 검증한다.

## 산출물

- `NodeGenerator.gd`
- `RewardResolver.gd`
- `RunProgression.gd`
- `NodeTable.json`
- mini-run telemetry report

## TDD 대상

- 노멀 노드 1개는 항상 후보에 포함된다.
- 랜덤 노드 후보는 seed로 재현된다.
- 보상 수량은 1~5개 가중치 테이블을 따른다.
- 클리어한 노드의 취약 파동과 관련된 보상 가중치가 증가한다.
- 스테이지가 오르면 마력장/체력/시간 압박이 상승한다.

## 완료 기준

- 3스테이지 미니 런을 자동으로 20회 실행할 수 있다.
- clear/time_over 분포, 적중률, 큐 누수율, 수리 횟수가 리포트된다.
- M0 재설계 게이트에서 유지/폐기할 도메인 계약을 판단할 수 있다.
