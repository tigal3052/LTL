# P3. 백팩과 방해 요소 슬라이스

## 목표

장비(유물) 배치가 전투 효율·리듬에 실제로 영향을 주고, 동결·파손 같은 방해 요소가 큐 생성과 진행을 흔드는지 검증한다.

## 산출물

- `InventoryModel.gd` (프로토타입 단계에서는 `app-LTL/src/domain/inventory-model.js`에 동일 계약으로 구현)
- `HazardModel.gd` (프로토타입 단계에서는 `app-LTL/src/domain/hazard-model.js`)
- 유물 도형·쿨타임·시너지 정의 데이터 (`artifact-table.json` 등)
- `RunSimulator`와 인벤토리 에너지 생성 경로 연동 (`run-simulator.js` 등)
- 방해 요소 1~2종(예: 동결 tick 지연, 파손 후 수리 전까지 생성 중단)
- `app-LTL/tests/p3_backpack_hazard.test.js` 및 필요 시 `app-LTL/public/index.html` 연동

## TDD 대상

- 8x8(또는 고정 크기) 그리드에 폴리오미노 유물을 배치·회전할 때 겹침·경계 밖 배치가 거절된다.
- 인접 동색 유물에 따른 시너지(쿨다운 단축 등)가 결정적으로 계산된다.
- 동결 적용 시 일정 tick 동안 해당 유물에서 에너지가 생성되지 않는다.
- 파손 시 생성이 멈추고, 수리 후 정상 생성으로 돌아온다.
- 시뮬레이터가 정적 큐 프리필만이 아니라 인벤토리 tick에 따른 생성 결과를 큐에 반영한다.

## 완료 기준

- 장비 배치 2개 이상 조합에서 초당(또는 tick당) 에너지 생성량 차이가 테스트 또는 로그로 확인된다.
- 방해 요소 적용 시 클리어·실패까지의 시간이 무방해 조건 대비 늘어나는 케이스가 테스트로 고정된다.
- `repair_count`·방해 요소 발생·수리와 같은 관측값이 리플레이/텔레메트리 또는 단위 테스트로 검증 가능하다.
- P2에서 정의한 조작·전투 루프와 모순 없이 동일 seed·input log 재현 경로가 유지된다.
