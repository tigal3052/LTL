# Tech Debt Tracker

> 구현 중 발견한 기술 부채를 기록한다.
> 바로 고치지 못하더라도 남겨야 다음 계획과 우선순위에 반영할 수 있다.

---

## Severity Guide

| Level | Meaning | Action |
|-------|---------|--------|
| Critical | 장애, 보안, 데이터 손실 위험 | 현재 작업 또는 즉시 후속 작업으로 처리 |
| Major | 확장 방해, 버그 유발, 개발 속도 저하 | 다음 마일스톤 전 계획 수립 |
| Minor | 가독성, 구조 정리, 일관성 문제 | 관련 영역 작업 시 함께 개선 |

---

## Rules

1. 기술 부채를 발견하면 삭제하지 말고 먼저 기록한다.
2. 현재 범위에 포함할지, 후속 작업으로 넘길지 결정한다.
3. 완료되면 해결 완료 섹션으로 이동하고 해결 방법을 남긴다.
4. 반복된 리뷰 실패 패턴은 자동화 후보로 표시한다.
