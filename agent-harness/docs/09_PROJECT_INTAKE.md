## 9. Acceptance Criteria

- [ ] 사용자가 무엇을 할 수 있게 되는지 측정 가능하게 적었다
- [ ] 실패 조건 또는 제외 범위를 적었다
- [ ] 테스트/검증 방법이 적혀 있다
- [ ] 출시 가능 기준이 명확하다

예시:

```text
- 신규 사용자는 3분 이내에 첫 작업을 완료할 수 있다.
- 핵심 API의 p95 응답시간은 300ms 이하를 유지한다.
- 운영자는 장애 발생 5분 내 원인을 파악할 수 있는 로그를 확인할 수 있다.
```
