# Comment Implementation Rules

이 문서는 `LTL-harness/00_AGENTS.md`, `agent-harness/00_AGENTS.md`가 공통으로 참조하는 주석-구현 규칙 SoT다.

## 핵심 규칙

- 함수 본문 안의 실행 주석은 반드시 바로 아래의 연속된 코드 블록 하나만 설명해야 한다.
- 함수 본문 안에서는 `주석A -> 주석A 구현 -> 주석B -> 주석B 구현` 순서만 허용한다.
- `주석A -> 주석B -> 주석A 구현 -> 주석B 구현` 형태는 금지한다.
- 두 개 이상의 실행 주석이 같은 코드 블록을 함께 설명하면 안 된다.
- 한 실행 주석이 여러 문장이나 여러 분기를 동시에 설명하면 안 된다.
- 어떤 주석이 바로 아래 한 덩어리 코드로 번역되지 않으면 그 주석은 실행 주석이 아니다.

## 허용되는 주석 종류

- 계약 헤더 주석: 파일, 모듈, 함수 입구에서 입력, 출력, 경계, 금지사항을 설명하는 주석.
- 직접 구현 주석: 바로 아래 코드 블록이 그 주석의 일을 직접 수행함을 설명하는 주석.
- 호출 중계 주석: 바로 아래 코드가 다른 함수 호출 하나로 일을 위임함을 설명하는 주석.

## 배치 규칙

- 계약 헤더 주석은 함수 본문 바깥이나 함수 시작부에 묶어서 둘 수 있다.
- 실행 주석은 함수 본문 안에서만 사용하고, 항상 자신이 설명하는 코드 바로 위에 둔다.
- 실행 주석과 구현 사이에는 다른 실행 주석, 빈 설명 문단, unrelated 선언을 끼워 넣지 않는다.
- 실행 주석 하나가 필요로 하는 코드가 길어지면 같은 코드 블록으로 유지하거나 더 작은 helper로 분리한다.
- 여러 실행 주석이 하나의 큰 블록을 공유하려고 하면 블록을 쪼개거나 helper를 추출한다.

## 수정 원칙

- 기존 주석 스캐폴드가 있는 파일은 전체 삭제 후 재작성하지 말고 필요한 위치만 부분 수정한다.
- 기존 주석 문장을 구현 편의 때문에 임의 삭제, 요약, 합치기 하지 않는다.
- 구현이 주석 배치를 따라가기 어렵다면 주석을 뭉개지 말고 코드 구조를 먼저 바꾼다.
- 실행 주석을 둘 연속 배치하고 싶은 유혹이 생기면 코드를 쓰기 전에 먼저 블록을 쪼갠다.
